### Key Differences Encountered

C, C++, and Python are _general-purpose programming languages_ used to implement logic and computational behavior.

HTML and CSS are _declarative languages_ used to structure and style content for display.

- HTML is a markup language.
- CSS is a stylesheet language.

These differences required a significant adjustment in learning approach.

### Adjustments Made

**1. Mindset Shift**

- CSS is declarative and cascading in nature. Instead of writing step-by-step logic, you describe *what* an element should look like, not *how* to compute it.

- Also, understanding layout flow, box models, inheritance, and specificity required a new way of thinking.

**2. Syntax Precision**

- In C/C++, small syntax errors can cause compilation failures or crashes due to strict typing and memory management.

- HTML and CSS are more forgiving:
    - Invalid rules are ignored.
    - Missing tags may be auto-corrected by browsers.
    - Styling conflicts are resolved via specificity and cascade order.

This flexibility required careful debugging discipline to maintain clean structure.
